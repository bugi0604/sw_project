# 1. Introduction

Ren'Py Game Translator는 Ren'Py 엔진으로 제작된 비주얼 노벨 게임의 번역 과정을 자동화하기 위한 시스템이다.

기존 Ren'Py 게임의 번역은 `.rpy` 스크립트 파일을 직접 수정하는 방식으로 진행된다. 이러한 방식은 번역 과정에서 변수나 태그가 손상될 가능성이 높으며, 스크립트 구조가 변경되어 게임 실행 오류가 발생할 수 있다.

본 시스템은 이러한 문제를 해결하기 위하여 Ren'Py 스크립트를 자동으로 분석하고, 번역 가능한 텍스트를 추출한 뒤 외부 번역 API를 통해 자동 번역을 수행한다.

또한 번역된 결과를 원본 스크립트 구조에 맞게 재삽입하여 새로운 `.rpy` 파일을 생성하며, 사용자는 번역 결과를 검토하고 직접 수정할 수 있다.

본 Design 단계에서는 Analysis 단계에서 정의된 기능 요구사항을 기반으로 실제 구현을 위한 클래스 구조, 객체 간 상호작용, 상태 변화 및 구현 환경을 설계한다.

# Class Diagram Explanation

시스템은 사용자가 프로젝트를 업로드하면 ProjectManager가 프로젝트를 관리하고 ScriptParser가 스크립트를 분석한다.

분석된 텍스트는 TranslationManager를 통해 TranslationAPI로 전달되며 자동 번역이 수행된다.

번역 결과는 TranslationResult에 저장되며 사용자는 TranslationEditor를 이용하여 번역 결과를 수정할 수 있다.

최종적으로 FileGenerator가 번역된 내용을 원본 스크립트 구조에 재삽입하여 새로운 `.rpy` 파일을 생성한다.

시스템 전체에서 발생하는 예외 상황은 ErrorHandler가 처리한다.

# 4. State Machine Diagram

본 장에서는 Ren'Py Game Translator 시스템의 상태 변화 과정을 설명한다.

시스템은 사용자의 입력과 내부 처리 결과에 따라 상태가 전환되며, 각 상태는 번역 작업의 진행 단계를 나타낸다.

---

## 4.1 Client State Machine

### States

#### Launch System

프로그램이 실행된 상태이다.

#### Upload Project

사용자가 Ren'Py 프로젝트 또는 `.rpy` 파일을 업로드하는 상태이다.

#### Configure Translation

사용자가 번역 언어와 옵션을 설정하는 상태이다.

#### Wait Analysis Result

시스템이 스크립트 분석 결과를 기다리는 상태이다.

#### Review Translation

사용자가 번역 결과를 검토하는 상태이다.

#### Edit Translation

사용자가 번역 결과를 수정하는 상태이다.

#### Generate File

최종 번역 파일을 생성하는 상태이다.

#### Download Result

생성된 번역 파일을 다운로드하는 상태이다.

#### Error State

오류가 발생하여 사용자에게 오류 메시지를 출력하는 상태이다.

---

## 4.2 Client State Transition

1. Launch System → Upload Project
2. Upload Project → Configure Translation
3. Configure Translation → Wait Analysis Result
4. Wait Analysis Result → Review Translation
5. Review Translation → Edit Translation
6. Edit Translation → Review Translation
7. Review Translation → Generate File
8. Generate File → Download Result
9. 모든 상태 → Error State

---

## 4.3 Server State Machine

본 시스템은 별도의 서버를 운영하지 않지만, 번역 API 및 내부 처리 모듈을 서버 역할로 간주하여 상태를 정의한다.

### States

#### Idle

사용자의 요청을 기다리는 상태이다.

#### Load Project

프로젝트 데이터를 읽는 상태이다.

#### Parse Script

스크립트를 분석하는 상태이다.

#### Extract Text

번역 가능한 텍스트를 추출하는 상태이다.

#### Request Translation

번역 API에 요청을 전송하는 상태이다.

#### Receive Translation

번역 API로부터 결과를 수신하는 상태이다.

#### Generate Output

최종 번역 파일을 생성하는 상태이다.

#### Save Output

생성된 파일을 저장하는 상태이다.

#### Error Handling

오류를 처리하는 상태이다.

---

## 4.4 Server State Transition

1. Idle → Load Project
2. Load Project → Parse Script
3. Parse Script → Extract Text
4. Extract Text → Request Translation
5. Request Translation → Receive Translation
6. Receive Translation → Generate Output
7. Generate Output → Save Output
8. Save Output → Idle
9. 모든 상태 → Error Handling

---

## State Machine Description

사용자가 프로젝트를 업로드하면 시스템은 스크립트를 분석하고 번역 가능한 텍스트를 추출한다.

이후 번역 API를 통해 자동 번역을 수행하고 결과를 사용자에게 제공한다.

사용자는 번역 결과를 검토 및 수정할 수 있으며, 최종적으로 새로운 `.rpy` 파일을 생성하고 다운로드할 수 있다.

각 단계에서 오류가 발생할 경우 Error State 또는 Error Handling 상태로 전환된다.

- 번역 수행 : 15초 이내
- 파일 생성 : 10초 이내

### Reliability

- 변수 및 태그 손상 방지
- 원본 스크립트 구조 유지
- 오류 발생 시 안정적인 복구

### Usability

- 직관적인 사용자 인터페이스 제공
- 번역 결과 비교 기능 제공
- 번역 수정 기능 제공

### Maintainability

- 모듈화된 클래스 구조 사용
- API 교체가 용이한 구조 제공

# 6. Glossary

| Term | Description |
|--------|--------|
| Ren'Py | 비주얼 노벨 제작을 위한 오픈소스 게임 엔진 |
| .rpy File | Ren'Py에서 사용하는 스크립트 파일 |
| Script Parser | 스크립트를 분석하여 번역 가능한 텍스트를 추출하는 모듈 |
| Translation API | 자동 번역 기능을 제공하는 외부 서비스 |
| Translation Result | 자동 번역 결과 데이터 |
| Variable | 게임 내에서 값을 저장하는 요소 |
| Tag | 텍스트 스타일 및 기능을 정의하는 요소 |
| Localization | 게임을 다른 언어로 번역하는 과정 |
| Placeholder | 변수 및 태그를 보호하기 위한 임시 문자열 |
| State Machine Diagram | 시스템의 상태 변화를 표현하는 UML 다이어그램 |
| Sequence Diagram | 객체 간 메시지 흐름을 표현하는 UML 다이어그램 |
| Class Diagram | 클래스 구조와 관계를 표현하는 UML 다이어그램 |
| User | 시스템을 사용하는 개발자 또는 번역자 |
| Error Handler | 시스템 오류를 처리하는 모듈 |
