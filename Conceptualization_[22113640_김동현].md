1. Business purpose
   
1-1. 기본 개념:
Ren’Py 기반 비주얼 노벨 게임의 번역 과정을 자동화하여 개발자와 번역자의 작업 효율을 향상시키는 도구 개발

1-2. 문제 정의
Ren’Py 게임은 대사가 많고 스크립트 구조가 복잡해서 수작업 번역이 매우 비효율적임
번역 과정에서 문맥 손실, 변수 깨짐, 코드 오류가 자주 발생
기존 번역 도구는 Ren’Py 특화 기능이 부족함

1-3. 해결 방법
Ren’Py 스크립트(.rpy)를 자동 분석하여 대사만 추출
번역 API(예: 기계번역)와 연동하여 자동 번역 수행
변수, 태그, 분기 구조를 유지한 상태로 재삽입
번역 검수 및 수정 UI 제공

1-4. 가치 / 기대 효과
번역 시간 단축 (수작업 대비 크게 감소)
번역 오류 감소 (코드 안정성 유지)
개인 개발자 및 인디 게임 팀의 글로벌 시장 진출 지원
비전문 번역자도 쉽게 접근 가능

본 프로젝트는 Ren’Py 엔진으로 제작된 게임의 스크립트를 분석하여 번역 가능한 텍스트를 자동 추출하고, 이를 효율적으로 번역 및 재구성하는 시스템을 설계하는 것을 목적으로 한다.
이를 통해 번역 과정에서 발생하는 오류를 줄이고 작업 시간을 단축하며, 인디 개발자 및 소규모 팀이 글로벌 시장에 보다 쉽게 진출할 수 있도록 지원한다.

4. Concept of operation

본 시스템은 Ren’Py 기반 비주얼 노벨 게임의 번역 작업을 자동화하기 위한 도구로, 사용자가 게임 스크립트를 업로드하면 이를 분석하여 번역 가능한 텍스트를 추출하고,
외부 번역 API를 활용하여 자동 번역을 수행한 뒤 결과를 제공하는 것을 목표로 한다.
   
Step 1: 프로젝트 업로드

사용자는 번역할 Ren’Py 프로젝트 또는 .rpy 파일을 시스템에 업로드한다.

Step 2: 번역 설정

사용자는 목표 언어(예: 한국어 → 영어)를 선택하고 번역 옵션을 설정한다.

Step 3: 스크립트 분석

시스템은 업로드된 파일을 분석하여 대사, 내레이션 등 번역 가능한 텍스트를 추출한다.
이 과정에서 변수, 태그, 분기 구조는 유지된다.

Step 4: 자동 번역 수행

추출된 텍스트는 외부 번역 API로 전달되며, 번역 결과가 반환된다.

Step 5: 결과 확인 및 수정

사용자는 번역 결과를 확인하고 필요한 경우 직접 수정할 수 있다.

Step 6: 번역 파일 생성

시스템은 번역된 내용을 기존 구조에 맞게 재삽입하여 새로운 .rpy 파일을 생성한다.

Step 7: 결과 다운로드

사용자는 완성된 번역 파일을 다운로드하여 자신의 프로젝트에 적용한다.

예외 상황
번역 API 응답 실패 → 재시도 또는 오류 메시지 출력
스크립트 파싱 실패 → 파일 형식 오류 안내
일부 문장 번역 실패 → 원문 유지 및 표시


5. Problem statement
   
5-1. 문제 정의
Ren’Py 기반 비주얼 노벨 게임은 대사 중심의 콘텐츠 특성상 번역이 필수적이며, 특히 글로벌 시장 진출을 위해 다국어 지원이 요구된다.
그러나 현재 대부분의 번역 작업은 .rpy 스크립트 파일을 직접 수정하는 방식으로 이루어지고 있어 비효율적이고 오류 발생 가능성이 높다.

5-2. 기존 방식의 한계
스크립트 구조가 복잡하여 번역 과정에서 코드 손상이 발생할 수 있음
변수, 태그, 분기 구조를 잘못 수정할 경우 게임 실행 오류 발생
수작업 번역으로 인해 시간과 비용이 많이 소요됨
일반 번역 도구는 Ren’Py 구조를 고려하지 않아 활용이 어려움

5-3. 문제의 영향
개발자의 작업 부담 증가
번역 품질 저하 및 버그 발생 가능성 증가
소규모 개발자 및 인디 게임 팀의 다국어 지원 어려움
결과적으로 글로벌 시장 진출 장벽 상승

5-4. 해결 필요성
이러한 문제를 해결하기 위해서는 Ren’Py 스크립트 구조를 이해하고, 번역 과정에서 코드의 안정성을 유지하면서
효율적으로 텍스트를 처리할 수 있는 자동화된 번역 시스템이 필요하다.


6. Glossary

| 용어           | 정의                                     |
| ------------ | -------------------------------------- |
| Ren’Py       | 비주얼 노벨 게임 제작에 사용되는 오픈소스 게임 엔진          |
| .rpy 파일      | Ren’Py에서 사용하는 스크립트 파일로, 대사와 게임 로직이 포함됨 |
| 스크립트         | 게임의 대사, 내레이션, 이벤트 흐름 등을 정의한 코드         |
| 번역           | 특정 언어의 텍스트를 다른 언어로 변환하는 과정             |
| 자동 번역        | 번역 API를 활용하여 텍스트를 자동으로 번역하는 기능         |
| 번역 API       | 외부 서비스로, 입력된 텍스트를 다른 언어로 변환해주는 시스템     |
| 텍스트 추출       | 스크립트에서 번역 가능한 문장만 분리하는 과정              |
| 파싱 (Parsing) | 파일의 구조를 분석하여 의미 있는 데이터로 변환하는 과정        |
| 변수           | 게임 내에서 값이 저장되는 요소 (예: 이름, 상태 등)        |
| 태그           | 텍스트에 스타일이나 기능을 적용하는 코드 요소              |
| 번역 결과        | 번역 API 또는 사용자가 생성한 최종 번역 텍스트           |
| 번역 검수        | 번역 결과를 사람이 확인하고 수정하는 과정                |
| 출력 파일        | 번역 결과가 반영된 새로운 .rpy 파일                 |
| 사용자          | 시스템을 사용하는 게임 개발자 또는 번역자                |
| 시스템          | Ren’Py 번역 자동화 도구 전체를 의미                |
